7 ## <a name="summary"></a>Summary
8 
9 **Alloy** is a flexible game-engine and development kit under active
10 development written in C++.
11 
12 It is characterized by upholding a strict set of design-goals which
13 center around maintainability and readability through proper
14 software engineering practices. The overall goal for this project
15 is to be a well-rounded and easy-to-use, efficient game-engine.
16 
17 It is inspired by many great open-source projects that
18 have come before it, the complete list can be seen
19 [below](#inspirations).

40 ## <a name="features"></a>Features
41 
42 * [x] Written with Modern C++ (C++17)
43  * [x] Supports for `constexpr` wherever possible
44 * [x] Designed using componentized architecture (easier reuse)
45 * [x] Strong focus on readability and design
46 * [ ] Supports multiple render systems _(in progress)_
47  * [ ] OpenGL
48  * [ ] DirectX
49  * [ ] Vulkan
50 * [ ] Supports multiple audio systems _(in progress)_
51  * [ ] OpenAL
52  * [ ] FMOD
53 * [ ] Handles intersections of geometric objects
54 * [ ] Easily build scene graphs and skeletons
55 * [ ] Manages game objects using entity-component system
56 
57 ## <a name="components"></a>Components
58 
59 **Alloy** is designed using component-based architecture in order to promote
60 better logical design and reusability through code-sharing.
61 
62 The library and application components are detailed below:
63 
64 ### Libraries
65 
66 * [`Alloy::Core`](lib/alloy-core/README.md) \
67  The core library that the rest of **Alloy** is built off of
68 * [`Alloy::IO`](lib/alloy-io/README.md) \
69  The library for managing all IO tasks (filesystem, network, etc)
70 * [`Alloy::Render`](lib/alloy-render/README.md) \
71  The library for managing rendering tasks, and abstracting the
72  underlying rendering system (DX, OpenGL, etc)
73 * [`Alloy::Audio`](lib/alloy-audio/README.md) \
74  The library for managing the audio tasks, and abstracting the
75  underlying mixing system (FMOD, OpenAL, etc)
76 * [`Alloy::Engine`](lib/alloy-engine/README.md) \
77  The library for managing all engine details, including subsystem
78  initialization & coordination, entity-component management,
79  resource management, etc.

85 ## <a name="inspirations"></a>Inspirations
86 
87 **Alloy** was inspired by a variety of existing open-source projects.
88 Some of these projects are listed below:
89 
90 - [Godot](https://github.com/godotengine/godot)
91 - [Blade](https://github.com/crazii/blade)
92 - [Ogre](https://github.com/OGRECave/ogre/pulls)
93 - [Nomad](https://github.com/taurheim/NomadECS)
94 - [easy-engine](https://github.com/callebstrom/easy-engine)
95 
96 **Note:** _inspired_ does not mean that any code was taken without following
97 proper license attribution. It simply means that certain designs and concepts
98 may have been adjusted to align with what other existing projects are
99 already doing.
