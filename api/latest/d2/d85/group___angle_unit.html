This concept defines the required interface and semantics expected of an "angle_unit".

The AngleUnit concept is used to identify the unit definition of one full revolution value for a given angle. This allows consuming code, e.g. the basic_angle type, to calculate revolutions, wrapping, and angle conversions in a generic manner.

For type A to be AngleUnit, it must satisfy the following:

Provided

  • A - an AngleUnit type
  • r - an instance of core::real

the following expressions must be well-formed with the expected side-effects:

r = AngleUnit::one_revolution();

one_revolution() is a static function which returns the numeric representation of the value of a single revolution in that angular unit.

This function must be marked constexpr, and must not throw an exception.
