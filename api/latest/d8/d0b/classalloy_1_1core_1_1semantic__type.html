Detailed Description

template<typename, typename>
class alloy::core::semantic_type< typename, typename >

A semantic type used for adding type-safety around underlying types.

This class wraps underlying types and propagates their operators to allow simple access, while also preventing incomparable assignments from underlying types.

The intended purpose of this is to create unique semantic types from underlying base types, such as wrapping up a 'string_view' or integral values into something with a stronger meaning. It requires explicit construction from the underlying type, giving better readability at the site of construction.

For example:

using percent = semantic_type<int,struct percent_tag>;
auto p = percent{100};
// p += 1; -- doesn't work!
p += percent{1}; // works! 0 overhead.

This is recommended to be used on any standard types like strings/ints to ensure that their context doesn't get misused. It can even be used granularly to prevent bugs at compile-time, for example:

using hp_type = semantic_type<int,struct hp_tag>;
using mp_type = semantic_type<int,struct mp_tag>;
struct stat_component
{
hp_type hp;
mp_type mp;
};
auto s = stat_component{100,50};
// s.hp += s.mp; -- illegal! source of bug
s.hp += semantic_cast<hp_type>(s.mp); // forced-conversion -- context is!
// intentional at call-site
